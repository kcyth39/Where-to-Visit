## 4. 残り画面の文言提案（決定用）

トップと同じ口調（やさしい・ひらがな多め）。「URL」は使わず、共有＝「みんなにおくるリンク」、編集＝「あなた専用リンク」で統一（トップA案と整合）。

### 4-1. イベント詳細（`/e`・`/o`）

| 箇所 | 現状 | 提案 |
|---|---|---|
| 通知（作成直後・オーナー） | オーナー編集URLを保存してください。 | **あなた専用リンクだよ。なくさないように保存してね。** |
| 通知（保存後） | 保存しました。 | **ほぞんしました！** |
| URLラベル: 共有URL | 共有URL | **みんなにおくるリンク** |
| URLラベル: オーナー編集URL | オーナー編集URL | **あなた専用リンク** |
| コピーボタン | コピー / コピー済み | **コピー / コピーしたよ！**（or 現状維持） |

### 4-2. オーナー編集（`/o` オーナー時）※「オーナーメニュー」は廃止

**設計方針（2026-07-09 おしげさん）**: オーナーとゲストは**同一レイアウトを共有**する。機能差は「**お題・メモを編集できるのはオーナーだけ**」の一点のみ。独立した「オーナーメニュー」見出し・パネルは設けない。目的: 初見ゲストの戸惑いを減らし、**ヒエラルキーを感じさせない共通体験**にする。※「きめたいひと」という語はUIでは使わない。

| 箇所 | 現状 | 提案 |
|---|---|---|
| 「オーナーメニュー」見出し・独立パネル | あり（aside＋h2「オーナーメニュー」） | **廃止**。ゲストと同じ画面に、オーナーだけ小さな編集手段を添える |
| 所有の表示 | なし | **控えめに**。小さく「あなたは お題とメモをなおせます」＋淡い色/小アイコン |
| 編集手段 | 別パネルのフォーム | お題／メモの近くに小さな「**なおす**」。押すと編集でき、保存は「**ほぞん**」 |
| 入力ラベル | イベント名 / メモ | **お題** / メモ（トップと統一） |

- 実装影響: `EventView.tsx` の `aside.owner-panel`（h2「オーナーメニュー」）を、インライン＆控えめな編集手段に再構成（Codex）。共有/あなた専用リンク行はそのまま（あなた専用リンクはオーナー時のみ表示）。

### 4-3. 状態表示（I-1 の出し分け）

`SetupMessage` を状況別の見出しにする。

| 状況 | 現状 | 提案（見出し / 本文） |
|---|---|---|
| 404（ページなし） | 「設定を確認してください」／「ページが見つかりません。」 | **「ページが みつかりません」** ／ 「リンクがまちがっているかもしれません。もういちどたしかめてね。」 |
| イベント不明（無効リンク） | 「設定を確認してください」／取得エラー文 | **「お題が みつかりません」** ／ 「リンクがまちがっているか、なくなっているのかもしれません。」 |
| Supabase設定エラー（開発者向け） | 「設定を確認してください」／「.env に…」 | 見出しは**現状維持**（開発者向け）。本文の **`.env` → `.env.local`** に修正（I-4） |

### 4-4. 任意（I-7）

- コピー失敗時（HTTP/権限拒否）に「**コピーできませんでした**」を表示するか（堅牢性・任意）。

### 4-5. 表示名（participant.display_name）の使いどころ（U-4の前提）

- 画面に出るのは**本人が入力した名前そのもの**（例「たろう」）。「ゲスト」という単語ではない。
- 表示される場所: **評価マトリクス**（誰が○/−/×＝Slice 3）／**❤️・🌀の付与者展開**（Slice 5）／**コメントの名前**（Slice 5）。
- **Slice 1では表示名はどこにも表示されない**（作成時に名前を集めるのみ）。
- よって U-4 は実質「**名前入力欄のラベル**をどうするか」。候補: (a) 全員共通「おなまえ」／(b) 作成時「きめたいひと」・参加時「おなまえ」／(c)「ゲスト」を使う。無階層方針とは (a)(b) が親和的。
