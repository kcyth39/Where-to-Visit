## トーン方針（2026-07-09 改訂）

- **漢字を積極使用**（情報密度優先・スマホ表示崩れ回避）。
- ひらがな維持の例外: ブランド「きめる」系（きめのすけ／きめよう！／きめの すけざえもん）、属性表示ラベル（みんなでやること／たべたりのんだり／とまるところ／ほかのこと）。
- 操作系ボタンは通例表記に倣う（例: キャンセル）。
- 「きめたいひと」「オーナーメニュー」という語はUIで使わない。

## 重要な整合ルール（実装厳守）

- **表示ラベルは画面表示のみの変更**。DBに保存する属性の**enum値（`食事 / 宿泊 / アクティビティ / そのた`）は変更しない**。
- したがって [04_data-model.md](../04_data-model.md) / [ADR-0003](../adr/0003-evaluation-and-decision-logic.md) の属性定義・**既存migrationは不変**。変更は表示層のみ。

### 属性: 表示ラベル・順序・お題placeholder

**属性選択を先に置き**、選ばれた属性に応じて「お題」の placeholder を切り替える。

| 表示順 | enum値（保存・不変） | 表示ラベル（維持） | お題 placeholder |
|---|---|---|---|
| 1 | アクティビティ | みんなでやること | 例） 次回のイベント、週末のお出かけ先 など |
| 2 | 食事 | たべたりのんだり | 例）週末のディナー、明日の朝ごはん など |
| 3 | 宿泊 | とまるところ | 例）夏休みに泊まるホテル など |
| 4 | そのた | ほかのこと | 例）お世話になった人へのプレゼント、名前 など |

- 未選択時のお題 placeholder: **「まず『どんなこと？』を選んでね」**。

## 名前入力欄（確定・全画面共通）

- **ラベルは全員共通「お名前」**（U-4=a）。オーナー作成時もゲスト参加時も同じ入力体験。
- 画面に表示されるのは本人が入力した名前そのもの（例「たろう」）。「ゲスト」という語はユーザーに見せない。
- 表示名が実際に出るのは Slice 3以降（評価マトリクス／❤️・🌀付与者／コメント）。

## イベント詳細画面（`/e/{shareToken}`・`/o/{ownerToken}`）確定コピー

| 箇所 | 確定文言 |
|---|---|
| ブランド | きめのすけ |
| eyebrow | 属性の表示ラベル（みんなでやること 等・自動） |
| 通知（作成直後・オーナー） | **あなた専用リンクだよ。無くさないように保存してね。** |
| 通知（保存後） | **保存しました！** |
| URLラベル: 共有 | **みんなに送るリンク** |
| URLラベル: オーナー編集 | あなた専用リンク（維持・オーナー時のみ表示） |

### オーナー編集（同一画面方針・確定）

- **オーナーとゲストは同一レイアウト**。機能差は「お題・メモを編集できるのはオーナーだけ」の一点のみ。
- **「オーナーメニュー」独立見出し・パネルは廃止**。
- **所有表示（控えめ）**: 小さく「**あなたは お題とメモを直せます**」＋淡い色/小アイコン。
- **編集手段**: お題／メモの近くに小さな「**直す**」→ 編集 → 保存「**保存**」。
- 入力ラベルは「**お題**」「メモ」。

### コピーボタン（確定・全画面共通）

- 既定ラベル「コピー」。**成功時はチェックマーク（✓）に変化**（約1.8秒で戻る）。失敗時の文言は出さない。
