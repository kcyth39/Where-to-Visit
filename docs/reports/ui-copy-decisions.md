## トーン方針

- きめのすけの**やさしい・親しみやすい口調**。ひらがな多め（ただし子供っぽくなりすぎない・大人も使える塩梅に）。
- マスコット名の例示: 「きめの すけざえもん」（おなまえ欄の placeholder）。
- **「きめたいひと」という語はUI文言としては使わない**（2026-07-09 廃止）。所有（編集可否）は「お題とメモをなおせます」と機能で表す。

## 重要な整合ルール（実装厳守）

- **表示ラベルは画面表示のみの変更**。DBに保存する属性の**enum値（`食事 / 宿泊 / アクティビティ / そのた`）は変更しない**。
- したがって [04_data-model.md](../04_data-model.md) / [ADR-0003](../adr/0003-evaluation-and-decision-logic.md) の属性定義・**既存migrationは不変**。変更は表示層（`constants.ts` の表示ラベル、フォーム、ページ文言）のみ。

### 属性: 表示ラベル・順序・お題placeholder

**属性選択を先に置き**、選ばれた属性に応じて「お題」の placeholder を切り替える。

| 表示順 | enum値（保存・不変） | 表示ラベル | お題 placeholder |
|---|---|---|---|
| 1 | アクティビティ | **みんなでやること** | 例） 次回のイベント、週末のお出かけ先 など |
| 2 | 食事 | **たべたりのんだり** | 例）週末のディナー、明日のあさごはん など |
| 3 | 宿泊 | **とまるところ** | 例）夏休みに泊まるホテル など |
| 4 | そのた | **ほかのこと** | 例）お世話になった人へのプレゼント、名前 など |

- 未選択時のお題 placeholder は、属性選択を促す文言（例: 「まず『どんなこと？』をえらんでね」）を暫定とする。※要確認（下記）。

### オーナー編集（同一画面方針・確定）

- **オーナーとゲストは同一レイアウト**。機能差は「お題・メモを編集できるのはオーナーだけ」の一点のみ。
- **「オーナーメニュー」独立見出し・パネルは廃止**。
- **所有表示（控えめ）**: 小さく「**あなたは お題とメモをなおせます**」＋淡い色/小アイコン。※「きめたいひと」という語はUIでは使わない。
- **編集手段**: お題／メモの近くに小さな「**なおす**」→ 編集 → 保存「**ほぞん**」。
- 入力ラベルは「**お題**」「メモ」（トップと統一）。

### コピーボタン（確定・全画面共通）

- 既定ラベル「**コピー**」。**成功時はチェックマーク（✓）に変化**（Claude風。約1.8秒で戻る）。
- **失敗時の文言は出さない**（成功の合図＝チェックマークのみ）。

## Codex 実装タスク（Slice 2 着手前・UI文言修正）

**表示層のみ。DB enum値・データモデル・既存migrationは変更しない。**
