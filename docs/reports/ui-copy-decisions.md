## トーン方針（2026-07-09 改訂）

- **漢字を積極使用**（情報密度優先・スマホ表示崩れ回避）。
- ひらがな維持の例外: ブランド「きめる」系（きめのすけ／きめよう！／きめの すけざえもん）。※旧「属性表示ラベル」は属性撤廃（ADR-0005）に伴い対象外。
- 操作系ボタンは通例表記に倣う（例: キャンセル）。
- 「きめたいひと」「オーナーメニュー」という語はUIで使わない。

## 重要な整合ルール（実装厳守）

> **現行UIでは「お題」「メモ」を入力ラベルとして使わない。** 「きめること」は`Event.title`、「つたえておきたいこと」は任意の`Event.memo`へ保存する。過去の改訂履歴や撤廃済み仕様に残る旧語は履歴として扱い、現行コピーへ適用しない。
>
> **⚠ ADR-0005（2026-07-10）で「属性」を撤廃。** 以下に残る属性関連（**属性選択・属性表示ラベル・属性連動のお題placeholder・イベント詳細 eyebrow の属性表示**）は**撤去対象**。お題placeholder は**汎用**にする。属性関連の記述は履歴として残すが、実際の撤去は Codex「属性最小撤去」タスクで行う。判断基準（Criterion）の実装は Slice 5。
> 漢字優先化改訂（下記）は属性に依存しない部分は有効。

## オーナー初期セットアップ（ADR-0007）

Event作成後、Event情報の下に導入文「**お名前と候補を入れたら、さあ、きめましょう！**」と2つの入力セクションを表示する。

| 手順 | タイトル | 説明文 |
|---|---|---|
| 1 | **お名前を入れる** | **ここで選んだ名前が、候補や回答の名義になります。** |
| 2 | **候補の追加** | **候補名だけでも、リンクだけでも追加できます。** |

- 名前確定後も同じ画面に残り、Candidate追加へ続く。追加済みCandidateはタイトル優先、タイトルが空ならURLを作成順で表示する。
- 候補追加フォームの入力ラベルは「**候補名**」とし、候補名inputにplaceholderは置かない。候補名・リンク・追加buttonはデスクトップ／モバイルとも別行にする。
- 「3. スタート」の見出しは置かない。「**さあ、きめよう！**」後は同じ画面を初回共有ステップへ切り替え、中央に「**みんなに送るリンク**」とコピーbuttonを表示する。
- 初回共有ステップの下に「**わたしの意見を入力**」を置き、同じタブのowner候補一覧ダッシュボードへ進む。
- 共有URL「みんなに送るリンク」とowner URL「あなた専用リンク」は候補一覧ダッシュボードのCandidate追加欄の下にも置く。
- きめること・つたえておきたいことの近くに小さな「直す」を置き、独立したオーナーパネルを作らない。

## ゲスト名前選択（ADR-0006 / ADR-0007）

- トップのEvent作成画面とCandidate追加フォームにお名前欄を置かない。
- 未選択ゲストには見出し「**あなたのお名前**」、既存名の選択肢、その直下のラベル「**直接入力**」と入力欄だけを表示する。
- 「まだ選んでいません」「回答者」「回答する人」「または新しいお名前」は表示しない。
- 既存名を選ぶと直接入力欄へ同じ名前を反映し、その行を選択して候補一覧へ進む。新名確定後も候補一覧へ進む。
- 候補編集の全回答者行は閲覧専用とし、○ / − / ×、❤️、🌀、コメントの編集controlは候補情報の上部へ集約する。
- 「ゲスト」「本人認証」「参加登録」という語をユーザーへ見せない。
- 同名時は同じ人か確認し、別人なら異なる名前の再入力を求める。
- 選択中回答者は候補編集で「**〇〇として判断中**」と表示する。「反応中」は使わない。
- 同名確認、名前変更、回答者削除の最終copyは、実装後の375px / 1366px実画面確認で調整する。構造と挙動は[ADR-0006](../adr/0006-collaborative-response-row-model.md)、[ADR-0007](../adr/0007-event-views-and-criterion-feedback.md)、[実装仕様](collaborative-response-row-spec-draft-2026-07-11.md)を正とする。

### オーナー編集（共通情報階層・確定）

- オーナー初期セットアップ後の候補一覧・候補編集はゲストと同じ情報階層を用いる。機能差は「きめること・つたえておきたいことを編集できるのはオーナーだけ」の一点のみ。
- 別ブラウザ・後日のowner URL再訪は、回答者未選択でも初期セットアップを再表示せず候補一覧を表示する。個人名義操作時だけ名前選択へ進む。
- **「オーナーメニュー」独立見出し・パネルは廃止**。
- **編集手段**: きめること／つたえておきたいことの近くに小さな「**直す**」→ 編集 → 保存「**保存**」。
- 入力ラベルは「**きめること**」「**つたえておきたいこと（任意）**」。

### 候補一覧ダッシュボード（ADR-0007）

- Candidate追加フォームにお名前欄を置かない。
- Eventのきめることとつたえておきたいことを画面上部に表示する。固定表示はしない。
- Candidate名の横に`⭕️ / ➖ / ❌`を別々のchipとして表示し、それぞれの件数を明示する。`➖`はneutral Vote行だけを数え、unratedを含めない。
- Candidate名の下にURL、その下に追加時期と提案者、その下に小さな❤️ / 🌀合計を表示する。
- Candidate作成時刻は「1時間以内に追加 / N時間前に追加 / N日前に追加」とする。クライアント時計より未来の場合は「1時間以内に追加」へclampする。
- 最終候補状態は`clear / discussion / fallback / none`のsemantic tokenを用いるが、「○最多・×なし」等の可視説明ラベルは表示しない。
- exact colorと評価chipの見え方は実画面確認後に確定する。

### 候補編集（ADR-0007）

- Candidate名・URL・追加時期・提案者の下に、サマリーと同じ○ / − / ×、❤️ / 🌀を配置する。❤️ / 🌀は同じ判断基準dialogを開く。
- 候補タイル内の評価control直下に選択中回答者の「**コメント**」入力欄と「**保存**」buttonを置く。
- 「**みんなの判断**」は全回答者の評価、判断基準別❤️ / 🌀、コメント全文を読む一覧とし、行内編集や行clickによる回答者変更を行わない。
- 詳細編集menuは「**候補内容の編集**」「**❤️／🌀反応項目の編集**」「**判断者名の変更／削除**」とする。候補内容だけを＋／−付き開閉UIからインラインで開き、反応項目編集と判断者名の変更／削除はmodalで開く。modal導線2件はデスクトップで同一行に並べて文言を改行せず、モバイルで横幅不足時だけボタン単位・文言とも折り返せる。反応項目編集は既存一覧の下に追加buttonを置き、追加・label変更・削除を扱う。判断者名の変更／削除は現在名を直接編集できるinputと、変更・キャンセル・右端の削除を同一modalに置く。
- Candidate単位の常設単一🌀は表示しない。
- 同名確認、変更確認、削除確認は一度に1画面だけを表示し、確認画面を並べない。

### コピーボタン（確定・全画面共通）

- 既定ラベル「コピー」。**成功時はチェックマーク（✓）に変化**（約1.8秒で戻る）。失敗時の文言は出さない。
