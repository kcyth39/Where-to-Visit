# ADR-0001: 事業化 Go 判断

- **ステータス:** Accepted
- **日付:** 2026-07-08

## コンテキスト

Phase 0（構想）として、リーンキャンバス（[01_lean-canvas.md](../01_lean-canvas.md)）と競合分析（[02_competitive.md](../02_competitive.md)）を整備した。複数人での意思決定支援サービス「きめのすけ」について、事業として着手するか（Go / No-Go / ピボット）を判断する必要がある。

判断材料の要点:

- 独自価値が「×（絶対ヤダ）が確定候補を保留させる構造」に集約され、既存競合（LINE投票 / Googleマップ グループプランニング / 多数決さん系 / 旅行計画アプリ）が扱えない領域が明確。
- 日程調整ツール（調整さん等）は非競合として除外でき、対象領域の重複が少ない。
- 収益は広告先行型で開始し、将来オーナー向けプレミアム（広告なし + AI解説）を追加する段階的モデルが描けている。
- 無料枠での開始が見込め、初期コストが低い。

## 決定

**きめのすけの事業化を Go とする。** 広告先行型の方針で MVP を開発し、Phase 1（要件定義）へ進む。

## 影響

- Phase 1（要件定義）に着手する。正本は `docs/` 配下とする。
- **ドメインは取得済み: `kimenosuke.com` / `kimenosuke.jp` / `kimenosuke.net` の3つを確保。**
- **primary（正）ドメインは `kimenosuke.com` に確定**（2026-07-08）。`kimenosuke.jp` / `kimenosuke.net` は `kimenosuke.com` へリダイレクトする。
- サービス名は「きめのすけ」を前提に以降のドキュメントを整備する。
- 収益・プレミアム（AI解説）は MVP 外。AI解説実装時は Anthropic API（Pro とは別契約・従量課金）が必要になる点を予算計画に含める。
