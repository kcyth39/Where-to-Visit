# ADR-0003: 評価モデルと確定ロジック

- **ステータス:** Accepted
- **日付:** 2026-07-07
- **関連:** [open-questions.md](open-questions.md)（B1〜B4 / B6〜B8 / N1 / N2）、権限は [ADR-0004](0004-permission-model.md)

## コンテキスト

きめのすけの中核は「複数人で1つに決める」ための評価モデルと確定判定である。単純な多数決では少数派の強い反対（拒否権）を扱えないため、評価の種類・確定の判定条件・各評価の可視性を厳密に定義する必要がある。本 ADR は Blocker（B1〜B4, B6〜B8）と否定的意見表明（N1, N2）の決定を確定させる。

## 決定

### 評価モデル

| 評価 | 役割 | デフォルト | 判定への関与 | 備考 |
|---|---|---|---|---|
| ○ | 総合評価「これがいい」 | −（○ではない） | **関与する** | 確定候補の判定に使う |
| − | 総合評価「うーん」（中間） | **−（初期値）** | 関与しない | 未評価と − は区別しない。中間スコアを持たない |
| × | 総合評価「絶対ヤダ」（拒否権） | −（×ではない） | **関与する** | 1つでもあれば確定候補から除外 |
| ❤️ | 非決定の**ポジ**意見（価格・属性別） | 無し | 関与しない | 温度感の補足。B系判定には不関与 |
| 🌀 | 非決定の**ネガ**懸念（単一） | 無し | 関与しない | 却下ではない軽い懸念の表明 |
| コメント | 補足テキスト | 無し | 関与しない | 任意。入力を促さない |

補足規則:

- 決定に関与するのは **総合評価 ○/−/× のみ**（B1）。❤️・🌀・コメントは判定に不関与。
- デフォルトは −。**未評価と能動的な − は、内部データ・表示のいずれにおいても区別しない**（判別ロジックを持たない）（B8）。
- 未参加者は評価表に**非在**（行として現れない）（B8）。
- − は中間スコアを持たない（順位付けに使う数値ではない）（B8）。

#### 属性別の有効❤️・🌀

| 属性 | 有効な ❤️ type | 🌀 |
|---|---|---|
| 食事 | price / taste / facility / place | 有効 |
| 宿泊 | price / facility / place | 有効 |
| アクティビティ | price / interest | 有効 |
| そのた | interest | 有効 |

- ❤️ は上表の属性別 type に付与できる非決定のポジ意見。🌀 は**全属性で有効**な単一の非決定ネガ懸念（N2）。
- コメントは任意で、システムから入力を促さない（N1）。

### 確定ロジック

- サービスは**確定という行為（確定ボタン・ロック）を扱わない**。条件を満たした候補を**自動でハイライト表示するのみ**（B2）。
- 判定に使うのは **○の数と ×の有無のみ**。−・❤️・🌀 は判定に関与しない。
- **×が1つでもある候補は確定候補から除外**し、**イシュー化を視覚的に明示**する（B4）。「理由確認中」等のテキストは**使用しない**。除外中でも**ブラックアウト・非表示にはしない**。
- **○が1つ以上ある候補**が確定候補ハイライトの対象（B4）。
- ○数が同数（タイ）の場合は**並列表示**する（B3）。
- **確定状態を持たない = ロックしない**（B6）。○数・×有無の変化に応じてハイライトはリアルタイムに変動する。
- **落選（非ハイライト）候補もブラックアウト・非表示にせず**、常時閲覧・再検討できる。

### 可視性

| 評価 | 可視性 |
|---|---|
| ○ | **付与者を全員に公開**（誰が○を付けたか全員が分かる）（B7） |
| − | **付与者を全員に公開**（誰が−かも全員が分かる）（B7） |
| × | **付与者を全員に公開**（誰が×を付けたか全員が分かる。理由確認のため）（B7） |
| ❤️ / 🌀 | **付与者を公開**（集計数＋タップで付与者を展開） |

- ○・−・× は**いずれも付与者を全員に公開**し、参加者×候補のマトリクスで表示する（B7）。

## 影響

- 実装（Code）は、判定ロジックに ○数・×有無以外の要素（−・❤️・🌀・コメント）を混入させてはならない。
- ○・−・× の付与者、および ❤️・🌀 の付与者は、表示可能な形でデータモデルに保持する（[04_data-model.md](../04_data-model.md) で反映）。
- 「確定」概念がないため、UI は「確定候補ハイライト」と「イシュー化（×あり）」を視覚表現で区別する（テキスト『理由確認中』は使わない）。落選候補も常時可視。
- ×→− への変更権限・履歴の扱いは [ADR-0004](0004-permission-model.md) で定義する。
