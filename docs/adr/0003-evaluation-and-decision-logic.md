# ADR-0003: 評価モデルと最終候補表示

- **ステータス:** Accepted
- **日付:** 2026-07-08
- **最終改訂:** 2026-07-11（[ADR-0006](0006-collaborative-response-row-model.md)）
- **関連:** [ADR-0004](0004-permission-model.md) / [ADR-0005](0005-drop-attribute-dynamic-criteria.md) / [04_data-model](../04_data-model.md)

## コンテキスト

きめのすけの役割は、候補に対するみんなの意見を見える化し、グループが決めやすい状態を作ることである。単純多数決だけでは少数派の強い反対を扱えず、一方で×がある候補や○最多未満の安全な候補を全く強調しないと、議論や消去法に必要な情報を落とす。

本ADRは、総合評価、補助情報、3種類の最終候補表示を定義する。回答者行と未評価の扱いはADR-0006で改訂した。

## 決定

### 評価モデル

| 表示 | DB / 読取状態 | 判定への関与 | 備考 |
|---|---|---|---|
| 未評価 | Vote行なし / `unrated` | 関与しない | 能動−と表示でも区別する |
| ○ | `positive` | ○数へ加算 | 「これがいい」 |
| − | `neutral` | 関与しない | 能動的な中立。中間スコアなし |
| × | `veto` | ×有無へ加算 | 強い反対・議論の必要性 |
| ❤️ | Reaction行 | 関与しない | 判断基準ごとのポジティブ補助情報 |
| 🌀 | Concern行 | 関与しない | 全お題常設の単一懸念 |
| コメント | Comment行 | 関与しない | Candidate×Participantにつき現在値1件 |

- VoteはCandidate×Participantにつき最大1行で、`positive / neutral / veto`の現在値だけを持つ。
- Vote行なしは未評価とし、`neutral`行と区別する。
- 判定に使うのは○数と×有無だけ。未評価と−はどちらも0として扱う。
- Vote時刻は保存・表示目的で追加しない。
- ○ / − / ×、❤️、🌀の付与者は候補カード内の回答者行として公開する。

### 判断基準・❤️・🌀

- 属性は[ADR-0005](0005-drop-attribute-dynamic-criteria.md)で撤廃済み。
- ❤️はEvent単位のCriterionごとに付ける/付けないの2値。
- Criterionはデフォルト「興味ある？」、プリセット「価格どう？ / 雰囲気どう？ / 場所はどう？ / 色はどう？」、自由記述からなる共同編集リスト。
- 🌀はCriterionへ統合せず、全お題に常設する単一の懸念。
- Candidate全体の❤️はCandidate配下のReaction行数、🌀はConcern行数を単純合計する。同一回答者が複数Criterionへ付けた❤️は複数件として数える。
- ❤️、🌀、コメントは最終候補状態へ影響しない。

### 最終候補の3状態

候補ごとの○数を`P`、×数を`X`、全候補の○最多数を`M`とする。

| semantic state | 条件 | 意味 |
|---|---|---|
| `clear` | `M > 0`, `P = M`, `X = 0` | 議論なしで決めやすい最有力候補 |
| `discussion` | `M > 0`, `P = M`, `X > 0` | 人気は最多だが議論が必要な候補 |
| `fallback` | clearが0件で、`0 < P < M`, `X = 0`の候補群における○最多 | 消去法で残る安全な代替候補 |
| `none` | 上記以外、または`M = 0` | 通常表示 |

補足規則:

- 同じ条件・同じ○数の候補はすべて同じ状態で並列表示する。
- ○最多が同数で、Aが○5・×0、Bが○5・×1なら、Aはclear、Bはdiscussionとする。
- clearが1件以上ある場合、○最多未満の×なし候補をfallbackにしない。Aが○10・×0、Bが○8・×0ならAだけclear、Bはnone。
- clearがなく、Aが○10・×1、Bが○5・×0、Cが○1・×0ならAはdiscussion、Bはfallback、Cはnone。
- ×がある候補もブラックアウト・非表示にせず、×数と回答者を常時表示する。
- 状態はDBへ保存せず、完全読取モデルから毎回導出する。
- 色だけに依存せず、状態ラベルまたはアイコン等の非色情報を併用する。exact visualは実画面確認後に確定する。

### 確定行為を持たない

- 確定ボタン、確定済み状態、ロック、落選候補の非表示を追加しない。
- 評価変更のたびに現在の完全状態から再計算する。
- サービスが示すのは意見分布に基づく候補状態であり、最終決定そのものではない。

## 影響

- 実装は○数・×有無以外を3状態判定へ混入させてはならない。
- UIはraw Vote行の有無を独自解釈せず、`unrated / positive / neutral / veto`へ正規化した完全読取モデルを使う。
- 旧仕様の「Vote行なし＝−」「未評価と能動−を区別しない」「×ありはすべてハイライト対象外」「×ゼロ・○最多だけを単一カテゴリとして強調」はADR-0006によりSUPERSEDED。
- 権限は[ADR-0004](0004-permission-model.md)、物理schemaは[04_data-model](../04_data-model.md)を参照する。
