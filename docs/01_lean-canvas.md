## リーンキャンバス 9項目

| # | 項目 | 内容 |
|---|---|---|
| 1 | 課題（Problem） | ・複数人での「決め」が投票（多数決）だけでは終わらない<br>・少数派の強い反対（拒否権）や懸念を扱う仕組みがない<br>・既存ツールは登録・アカウントが前提で参加のハードルが高い |
| 2 | 顧客セグメント（Customer Segments） | ・家族/友人/同僚など少人数グループの幹事（オーナー）<br>・食事・宿泊・アクティビティ・そのた（買い物など）の行き先を複数人で決めたい人<br>・アーリーアダプター: 家族旅行の宿決め、友人との食事店決めをする幹事層 |
| 3 | 独自の価値提案（UVP） | **×（絶対ヤダ）が確定候補を保留させる構造**で「全員が許容できる決定」に到達する意思決定支援。登録不要で参加でき、ログインは任意 |
| 4 | ソリューション（Solution） | ・○/−/× の総合評価（デフォルト−）＋自動ハイライト<br>・×が付いた候補は確定候補から除外し、イシュー化を視覚的に明示（「理由確認中」等テキストは不使用）<br>・×→−は誰でも変更でき懸念解消を促す<br>・❤️（非決定ポジ）/🌀（非決定ネガ）/コメントで温度感を補足 |
| 5 | チャネル（Channels） | ・共有URL（口コミ・グループチャット経由の招待）<br>・note（AI3役体制での個人開発の実録記事）<br>・X（利用シーン別の短文投稿） |
| 6 | 収益の流れ（Revenue Streams） | ・**MVP: 無料 + 広告**（ディスプレイ広告）<br>・将来: オーナー向けプレミアム（広告なし + 候補へのAI解説）※MVP外 |
| 7 | コスト構造（Cost Structure） | ・ホスティング/DB（無料枠で開始想定）<br>・ドメイン維持費（kimenosuke 系）<br>・将来のAI解説は Anthropic API 従量課金（Pro とは別契約） |
| 8 | 主要指標（Key Metrics） | ・作成イベント数 / 参加者数<br>・**×解消フローの発生・解消率**（独自価値が使われているかの中核指標）<br>・確定候補ハイライト到達率、再訪率 |
| 9 | 圧倒的な優位性（Unfair Advantage） | ・拒否権（×）と懸念解消を意思決定の中核に据えた設計思想<br>・登録不要 × 任意ログインの低摩擦体験（調整さんのベストプラクティス流用）<br>・AI3役体制（人間=意思決定 / Chat / Cowork / Code）による高速な個人開発 |
