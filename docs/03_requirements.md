# 03 要件定義（きめのすけ）

作成日: 2026-07-08 / フェーズ: Phase 1（要件定義）
正本: 本書 + [ADR-0003](adr/0003-evaluation-and-decision-logic.md)（評価・確定）/ [ADR-0004](adr/0004-permission-model.md)（権限）。未決・保留は [adr/open-questions.md](adr/open-questions.md) に集約。

---

## 1. ユーザー種別

| 種別 | ログイン | 権限・特徴 | 識別 |
|---|---|---|---|
| オーナー | 不要 | お題作成者。お題名・メモの編集導線（独立「オーナーメニュー」パネルは廃止・同一画面内の控えめ表示）、オーナー編集URL（`owner_token`）を保持 | Cookie/トークンで判別 |
| ゲスト | 不要・登録不要 | 共有URLで参加。候補追加・評価・❤️・🌀・コメントが可能 | `guest_token`（Cookie/localStorage） |
| （MVP外）ログインユーザー | 要 | 端末横断のイベント一覧。会員登録は MVP 外 | — |

## 2. 機能要件（受け入れ条件 Given/When/Then）

### スライス1: お題作成と共有

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-1.1 お題作成 | **Given** トップ画面 **When** イベント名（お題・自由テキスト）・メモ（任意）を入力し作成（**属性選択はしない**・[ADR-0005](adr/0005-drop-attribute-dynamic-criteria.md)） **Then** イベントが作成され、共有URLが発行され、作成者がオーナーとして記録される。※判断基準（Criterion）の実装は **Slice 5**。デフォルト「興味ある？」の付与（seed）も **Slice 5 のお題作成時から**行う（**再push版ではseedしない**・既存イベントは Slice 5 で backfill） |
| AC-1.2 共有URL発行 | **Given** イベント作成完了 **When** 共有URLが発行される **Then** URLは推測困難なトークンを含み、URLを知る人のみアクセスでき、検索エンジンに非インデックス |
| AC-1.3 ゲスト未ログイン閲覧 | **Given** 未ログインのゲスト **When** 共有URLで開く **Then** ログインなしで閲覧でき、`guest_token` を Cookie 保持（同一ブラウザ再訪で識別）。※**評価（○/−/×）と自分の回答の確認・修正は Slice 3**（評価UIはSlice 3のため、Slice 1では扱わない） |
| AC-1.4 オーナー編集 | **Given** イベントを作成したブラウザ（トークン一致） **When** イベントを開く **Then** お題（イベント名）・メモの編集手段が表示される（トークン不一致では非表示・独立「オーナーメニュー」パネルは廃止、同一画面内の控えめ表示）。**編集確定時は「変更します、よろしいですか？」の確認を挟む** |
| AC-1.6 URLコピー | **Given** イベント詳細画面 **When** コピー操作 **Then** イベントURLをワンクリックでコピーできる |
| AC-1.7 オーナー編集URL発行 | **Given** イベント作成完了 **When** オーナー編集URL（`owner_token`）が発行される **Then** 共有URLとは別に発行され、保存を促す表示がなされる |
| AC-1.8 編集URLで権限回復 | **Given** Cookie 消失または別端末 **When** オーナー編集URLにアクセス **Then** お題・メモの編集導線を利用できる（権限回復） |

### スライス2: 候補管理

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-2.1 候補追加 | **Given** 共有URL/オーナー編集URLで開いている人 **When** タイトル または URL の**少なくとも一方**（両方も可）を入れて追加 **Then** 候補が追加される。**お名前入力は強制しない**（候補追加フォームの任意欄）。追加者のブラウザ（`guest_token`）の Participant を（無ければ）自動生成し、提案者（`created_by`）に自動設定する。**再追加時にお名前入力があれば自分の表示名を上書き更新**（専用の名前編集UIは作らない・既存候補の提案者名にも反映） |
| AC-2.2 候補表示 | **Given** 候補が存在 **When** 一覧を表示 **Then** **タイトルがあればタイトル、無ければURL**を表示（URLはリンク）。**提案者（お名前）**も表示し、**未設定は「ー」** |
| AC-2.3 初期評価 | **Given** 新規候補が追加された **When** 参加者から見る **Then** 評価はまだ存在しない（初期＝−の状態）。**評価操作UIは作らない**（評価・`votes` テーブルは Slice 3）。Slice 2 の検証は「候補・参加者・提案者が作られること」＋「評価UIが無いこと」。※「vote行なし＝−」のDBアサーションは `votes` 実装後（Slice 3）に行う |
| AC-2.4 候補編集（タイトル/URL） | **Given** URLを知る人（性善説・B案） **When** タイトルまたはURLを変更して確定 **Then** 誰でも編集でき、**要素ごとに「変更します、よろしいですか？」の確認**を経て確定。編集後も「タイトル or URL のどちらか一方は必須」を満たす |
| AC-2.5 候補削除 | **Given** URLを知る人（B案） **When** 候補を削除 **Then** 誰でも削除でき、**2段階の確認**（1回目「この候補を消しますか？」→2回目「本当によろしいですか？」・2回目はより強い警告色で文言・配色とも差別化）を挟み、物理削除＋カスケード削除される |
| AC-2.6 提案者の編集 | **Given** URLを知る人（性善説） **When** 提案者を**プルダウン（既存参加者＋「ー」）**で変更して確定 **Then** 誰でも変更でき、**「変更します、よろしいですか？」の確認**を経て確定。記入は強制しない |

> ※候補追加まわりに「**タイトルかリンクのどちらかでOK・候補はいつでも追加できます（未評価・未参加でも）**」を適切な位置に明示し、「どれも評価できず行き詰まる」状態を防ぐ。候補タイトルの placeholder は**汎用の確定文言「例）候補の名前 など」**（属性は撤廃・[ADR-0005](adr/0005-drop-attribute-dynamic-criteria.md)）。お題入力欄の placeholder は「例）週末どこ行く？ など」。

### スライス3: 総合評価

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-3.1 評価付与 | **Given** 候補一覧 **When** ○/−/× を候補ごとに付与 **Then** 1候補につき1評価で排他的に記録される |
| AC-3.2 デフォルト− | **Given** 未操作の候補 **When** 何もしない **Then** 評価は「−」のまま（未評価＝−） |
| AC-3.3 可視性 | **Given** 評価が付与されている **When** 一覧を表示 **Then** ○・−・× いずれも付与者を全員に公開する（参加者×候補のマトリクス表示） |
| AC-3.4 ○集計 | **Given** 評価が付与されている **When** 集計を表示 **Then** 各候補の ○ 数が集計表示される |
| AC-3.5 評価修正 | **Given** 同一ブラウザの参加者 **When** 自分の評価を変更 **Then** 修正できる（性善説で誰でも変更可） |

- ※未評価と能動的な「−」は、内部データ・表示とも区別しない（判別ロジックを持たない）。

### スライス4: 確定ロジック

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-4.1 確定候補ハイライト | **Given** 候補と評価が存在 **When** 「×ゼロ・○1つ以上・○最多」を満たす **Then** 当該候補が自動ハイライトされる（確定という行為はない） |
| AC-4.2 ×でイシュー化 | **Given** ある候補に × が1つ以上ある **When** 判定される **Then** 確定候補から除外され、イシュー化が視覚的に明示される（「理由確認中」等のテキストは不使用、ブラックアウトもしない） |
| AC-4.3 ○タイ並列 | **Given** ○最多が同数の候補が複数 **When** ハイライト判定 **Then** 複数を並列ハイライトし、絞り込まない |
| AC-4.4 ×→−変更 | **Given** URLを知る参加者 **When** × を − に変更 **Then** 誰でも変更でき、履歴は残さない |
| AC-4.5 リアルタイム追随 | **Given** 評価が変更された **When** 再計算 **Then** 確定候補が常時再計算され、ロックはない |
| AC-4.6 全候補可視 | **Given** 落選（非ハイライト）候補 **When** 一覧を表示 **Then** ブラックアウト・非表示にせず、常時閲覧・再検討できる |

### スライス5: ❤️・🌀・コメント

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-5.0 判断基準UI（旧AC-1.5から移設） | **Given** お題 **When** 判断基準を表示 **Then** そのお題の**判断基準（Criterion）**ごとに ❤️ を付けられる（属性連動ではなく、イベント単位の共有リスト・共有URLを知る全員が追加/編集/削除可・参加不要）。デフォルト「興味ある？」＋プリセット＋自由記述・[ADR-0005](adr/0005-drop-attribute-dynamic-criteria.md) |
| ~~AC-5.1 価格❤️~~ | **廃止（ADR-0005）**。「価格」は判断基準のプリセット「価格どう？」として AC-5.2 に統合 |
| AC-5.2 判断基準❤️ | **Given** お題の判断基準（Criterion） **When** 候補にその基準の❤️を付け外し **Then** デフォルトなし・即時反映・非決定（決定に関与しない）。判断基準は**デフォルト「興味ある？」＋プリセット（価格どう？/雰囲気どう？/場所はどう？/色はどう？）＋自由記述**で、**共有URLを知る全員が誰でも追加/編集/削除可（参加不要・削除は2重確認）**・[ADR-0005](adr/0005-drop-attribute-dynamic-criteria.md) |
| AC-5.3 🌀 | **Given** 候補（全お題共通） **When** 🌀を付ける **Then** デフォルトなし・即時反映・非決定（×の手前の難色）。🌀は判断基準とは別の**常設の単一懸念** |
| AC-5.4 ❤️・🌀可視性 | **Given** ❤️・🌀が付与されている **When** 表示 **Then** 付与者を公開する（集計数＋タップで付与者を展開） |
| AC-5.5 コメント | **Given** 候補 **When** コメントを入力 **Then** 任意で投稿でき、システムは入力を促さない（テキスト入力極力不要の原則） |
| AC-5.6 コメント編集削除 | **Given** URLを知る参加者 **When** コメントを編集・削除 **Then** 誰でも可能（性善説） |

### マイイベント一覧（Cookieベース・MVP内）

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-6M | **Given** あるブラウザで作成/参加したイベント **When** 一覧を表示 **Then** Cookie/localStorage から一覧表示し、オーナー/参加者を区別する。Cookie 消失で一覧は消える（端末横断はしない） |

## 3. 非機能要件

| 区分 | 要件 |
|---|---|
| 対応環境 | モバイル・デスクトップ同格。モバイル制約を先に詰め、デスクトップはオーナー中心の編集性を追求 |
| セキュリティ・プライバシー | `share_token`/`owner_token` は推測困難。`noindex`＋`robots`。URLを知る人のみアクセス。広告・トラッキングのプライバシーポリシー記載・Cookie 同意 |
| データ・ライフサイクル | 無期限保存、上限なし（技術上限のみ）、イベント削除機能なし |
| 整合性・性能 | 同時編集は last-write-wins。評価・❤️・🌀 は即時反映 |
| 識別 | Cookie/localStorage（ログイン不要）、オーナー編集URLで権限回復 |

## 4. MVP境界

### In Scope

お題作成（自由テキスト・メモ・共有URL・オーナー編集URL／**属性なし**） / 候補管理 / 総合評価 / 確定候補・イシュー化 / 判断基準（Criterion）＋❤️・🌀 / コメント / URLコピー / マイイベント一覧（Cookieベース） / 広告実装 / モバイル・デスクトップ対応 / noindex・無期限保存

### Out of Scope（スコープ外）

- **認証・履歴**: ログイン機能・会員登録・端末横断のイベント一覧
- **収益・成長**: プレミアムプラン（AI解説）
- **機能拡張**: 通知 / 複数お題グルーピング / 外部AI相談プロンプト / ゲスト候補追加の制御 / 参加者並び替え / URLからの候補名称の自動導出（リリース後）
- **ライフサイクル**: イベント有効期限・アーカイブ / イベント削除機能
- **割り切り**: 複数端末からの完全な重複防止
- **持たない**: ×解消の履歴表示 / 🌀以外のスタンプ

## 5. 画面一覧

### 主要画面

| 画面 | 内容 |
|---|---|
| お題作成画面 | イベント名（お題・自由テキスト）・メモ → 作成 → 共有URL＋オーナー編集URL発行（**属性選択なし**） |
| イベント詳細画面（中核） | ヘッダ（お題名・メモ・オーナーは控えめな編集導線「直す/保存」・URLコピー／独立「オーナーメニュー」パネルは廃止） / 候補リスト（提案者表示・確定候補ハイライト・イシュー明示・落選候補も可視／評価・❤️（判断基準）・🌀・コメントは後続スライス） / 候補追加（お名前は任意） /（将来）AI相談プロンプトコピー |
| マイイベント一覧（Cookieベース） | そのブラウザで関わったイベント。オーナー/参加者を区別 |

### 主要UI要素

- お名前入力（任意・候補追加フォーム内） / 候補追加フォーム / オーナーの控えめな編集導線（「直す/保存」・独立パネルなし） / 削除確認ダイアログ（2段階・配色差） / 変更確認「変更します、よろしいですか？」 / URLコピー
- ※ログイン / 会員登録画面は MVP 外
