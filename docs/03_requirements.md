# 03 要件定義（きめのすけ）

作成日: 2026-07-08 / フェーズ: Phase 1（要件定義）
正本: 本書 + [ADR-0003](adr/0003-evaluation-and-decision-logic.md)（評価・確定）/ [ADR-0004](adr/0004-permission-model.md)（権限）。未決・保留は [adr/open-questions.md](adr/open-questions.md) に集約。

---

## 1. ユーザー種別

| 種別 | ログイン | 権限・特徴 | 識別 |
|---|---|---|---|
| オーナー | 不要 | お題作成者。イベント名・メモ編集権限、オーナーメニュー、オーナー編集URL（`owner_token`）を保持 | Cookie/トークンで判別 |
| ゲスト | 不要・登録不要 | 共有URLで参加。候補追加・評価・❤️・🌀・コメントが可能 | `guest_token`（Cookie/localStorage） |
| （MVP外）ログインユーザー | 要 | 端末横断のイベント一覧。会員登録は MVP 外 | — |

## 2. 機能要件（受け入れ条件 Given/When/Then）

### スライス1: お題作成と共有

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-1.1 お題作成 | **Given** トップ画面 **When** イベント名・メモ（任意）・属性（食事/宿泊/アクティビティ/そのた）を入力し作成 **Then** イベントが作成され、共有URLが発行され、作成者がオーナーとして記録される |
| AC-1.2 共有URL発行 | **Given** イベント作成完了 **When** 共有URLが発行される **Then** URLは推測困難なトークンを含み、URLを知る人のみアクセスでき、検索エンジンに非インデックス |
| AC-1.3 ゲスト閲覧・回答保持 | **Given** 未ログインのゲスト **When** 共有URLで閲覧・評価する **Then** ログインなしで閲覧・評価でき、`guest_token` を Cookie 保持。同一ブラウザ再訪時に自分の回答を確認・修正できる |
| AC-1.4 オーナーメニュー | **Given** イベントを作成したブラウザ（トークン一致） **When** イベントを開く **Then** イベント名・メモを編集するオーナーメニューが表示される（トークン不一致では非表示） |
| AC-1.5 属性別UI | **Given** 選択済みの属性 **When** 評価UIを表示 **Then** 属性に応じた ❤️ 種別が評価UIに表示される |
| AC-1.6 URLコピー | **Given** イベント詳細画面 **When** コピー操作 **Then** イベントURLをワンクリックでコピーできる |
| AC-1.7 オーナー編集URL発行 | **Given** イベント作成完了 **When** オーナー編集URL（`owner_token`）が発行される **Then** 共有URLとは別に発行され、保存を促す表示がなされる |
| AC-1.8 編集URLで権限回復 | **Given** Cookie 消失または別端末 **When** オーナー編集URLにアクセス **Then** オーナーメニューを利用できる（権限回復） |

### スライス2: 候補管理

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-2.1 候補追加 | **Given** オーナーまたはゲスト **When** タイトル（必須）＋URL（任意）で候補を追加 **Then** 候補が追加される。初回の能動アクション時に表示名を1回入力し、参加（Participant）が生成される |
| AC-2.2 候補表示 | **Given** 候補が存在 **When** 一覧を表示 **Then** タイトルとURL（あればリンク）が一覧表示される |
| AC-2.3 初期評価 | **Given** 新規候補が追加された **When** 参加者から見る **Then** 全参加者の初期評価が「−」である |
| AC-2.4 候補編集 | **Given** URLを知る参加者（性善説） **When** 候補を編集 **Then** 誰でも編集できる |
| AC-2.5 候補削除 | **Given** URLを知る参加者 **When** 候補を削除 **Then** 誰でも削除でき、2重確認ダイアログを挟み、物理削除＋カスケード削除される |

### スライス3: 総合評価

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-3.1 評価付与 | **Given** 候補一覧 **When** ○/−/× を候補ごとに付与 **Then** 1候補につき1評価で排他的に記録される |
| AC-3.2 デフォルト− | **Given** 未操作の候補 **When** 何もしない **Then** 評価は「−」のまま（未評価＝−） |
| AC-3.3 可視性 | **Given** 評価が付与されている **When** 一覧を表示 **Then** ○・−・× いずれも付与者を全員に公開する（参加者×候補のマトリクス表示） |
| AC-3.4 ○集計 | **Given** 評価が付与されている **When** 集計を表示 **Then** 各候補の ○ 数が集計表示される |
| AC-3.5 評価修正 | **Given** 同一ブラウザの参加者 **When** 自分の評価を変更 **Then** 修正できる（性善説で誰でも変更可） |

- ※未評価と能動的な「−」は、内部データ・表示とも区別しない（判別ロジックを持たない）。

### スライス4: 確定ロジック

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-4.1 確定候補ハイライト | **Given** 候補と評価が存在 **When** 「×ゼロ・○1つ以上・○最多」を満たす **Then** 当該候補が自動ハイライトされる（確定という行為はない） |
| AC-4.2 ×でイシュー化 | **Given** ある候補に × が1つ以上ある **When** 判定される **Then** 確定候補から除外され、イシュー化が視覚的に明示される（「理由確認中」等のテキストは不使用、ブラックアウトもしない） |
| AC-4.3 ○タイ並列 | **Given** ○最多が同数の候補が複数 **When** ハイライト判定 **Then** 複数を並列ハイライトし、絞り込まない |
| AC-4.4 ×→−変更 | **Given** URLを知る参加者 **When** × を − に変更 **Then** 誰でも変更でき、履歴は残さない |
| AC-4.5 リアルタイム追随 | **Given** 評価が変更された **When** 再計算 **Then** 確定候補が常時再計算され、ロックはない |
| AC-4.6 全候補可視 | **Given** 落選（非ハイライト）候補 **When** 一覧を表示 **Then** ブラックアウト・非表示にせず、常時閲覧・再検討できる |

### スライス5: ❤️・🌀・コメント

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-5.1 価格❤️ | **Given** 候補 **When** 価格❤️を付け外し **Then** デフォルトなし、即時反映され、決定には関与しない |
| AC-5.2 属性別❤️ | **Given** 選択属性 **When** 属性別❤️を付ける **Then** 有効type（食事=price/taste/facility/place、宿泊=price/facility/place、アクティビティ=price/interest、そのた=interest）を表示し、即時反映、非決定 |
| AC-5.3 🌀 | **Given** 全属性の候補 **When** 🌀を付ける **Then** デフォルトなし・即時反映・非決定（×の手前の難色） |
| AC-5.4 ❤️・🌀可視性 | **Given** ❤️・🌀が付与されている **When** 表示 **Then** 付与者を公開する（集計数＋タップで付与者を展開） |
| AC-5.5 コメント | **Given** 候補 **When** コメントを入力 **Then** 任意で投稿でき、システムは入力を促さない（テキスト入力極力不要の原則） |
| AC-5.6 コメント編集削除 | **Given** URLを知る参加者 **When** コメントを編集・削除 **Then** 誰でも可能（性善説） |

### マイイベント一覧（Cookieベース・MVP内）

| ID | 受け入れ条件（Given / When / Then） |
|---|---|
| AC-6M | **Given** あるブラウザで作成/参加したイベント **When** 一覧を表示 **Then** Cookie/localStorage から一覧表示し、オーナー/参加者を区別する。Cookie 消失で一覧は消える（端末横断はしない） |

## 3. 非機能要件

| 区分 | 要件 |
|---|---|
| 対応環境 | モバイル・デスクトップ同格。モバイル制約を先に詰め、デスクトップはオーナー中心の編集性を追求 |
| セキュリティ・プライバシー | `share_token`/`owner_token` は推測困難。`noindex`＋`robots`。URLを知る人のみアクセス。広告・トラッキングのプライバシーポリシー記載・Cookie 同意 |
| データ・ライフサイクル | 無期限保存、上限なし（技術上限のみ）、イベント削除機能なし |
| 整合性・性能 | 同時編集は last-write-wins。評価・❤️・🌀 は即時反映 |
| 識別 | Cookie/localStorage（ログイン不要）、オーナー編集URLで権限回復 |

## 4. MVP境界

### In Scope

お題作成（属性・メモ・共有URL・オーナー編集URL） / 候補管理 / 総合評価 / 確定候補・イシュー化 / ❤️・🌀 / コメント / URLコピー / マイイベント一覧（Cookieベース） / 広告実装 / モバイル・デスクトップ対応 / noindex・無期限保存

### Out of Scope（スコープ外）

- **認証・履歴**: ログイン機能・会員登録・端末横断のイベント一覧
- **収益・成長**: プレミアムプラン（AI解説）
- **機能拡張**: 通知 / 複数お題グルーピング / 外部AI相談プロンプト / ゲスト候補追加の制御 / 参加者並び替え
- **ライフサイクル**: イベント有効期限・アーカイブ / イベント削除機能
- **割り切り**: 複数端末からの完全な重複防止
- **持たない**: ×解消の履歴表示 / 🌀以外のスタンプ

## 5. 画面一覧

### 主要画面

| 画面 | 内容 |
|---|---|
| お題作成画面 | イベント名・メモ・属性選択 → 作成 → 共有URL＋オーナー編集URL発行 |
| イベント詳細画面（中核） | ヘッダ（名前・メモ・オーナーメニュー・URLコピー） / 候補リスト（評価・❤️・🌀・コメント、確定候補ハイライト・イシュー明示・落選候補も可視） / 参加者×候補の評価表マトリクス / 候補追加 /（将来）AI相談プロンプトコピー |
| マイイベント一覧（Cookieベース） | そのブラウザで関わったイベント。オーナー/参加者を区別 |

### 主要UI要素

- 表示名入力（初回能動アクション時） / 候補追加フォーム / オーナーメニュー / 削除確認ダイアログ（2重） / ❤️・🌀 付与者展開 / URLコピー
- ※ログイン / 会員登録画面は MVP 外
